## 3. Писане на функцията `predict`

Автоматизираният оценител (evaluator) разчита на стандартна входна точка. Някъде в избраните от вас клетки трябва да дефинирате функция с точното име `predict`.

### Пример: Текстова класификация
```python
# SUBMIT

import joblib
from sklearn.feature_extraction.text import TfidfVectorizer

# Предполага се, че тези файлове са предоставени в папката с датасета на задачата
vectorizer = joblib.load("vectorizer.pkl")
model = joblib.load("model.pkl")

def predict(inputs_list):
    """
    Вход: Списък (list) от текстови низове.
    Изход: Списък от целочислени прогнози.
    """
    features = vectorizer.transform(inputs_list)
    predictions = model.predict(features)
    
    # Уверете се, че връщате стандартен Python list
    return list(predictions)
```

## 5. Проследяване на опашката и гратисни периоди

### Обновявания в реално време
След като предадете решение, можете да проследите статуса му в историята на решенията:
* **Queued (В опашката):** Изчаква се наличен worker node.
* **Running (Изпълнява се):** Docker контейнерът изпълнява вашия код.
* **Evaluating (Оценява се):** Вашите прогнози се сравняват със скрития тестов датасет.
* **Completed / Failed (Завършено / Неуспешно):** Изпълнението приключи. Ако е неуспешно, кликнете, за да разгънете лога за грешки за подробен stack trace.

### Гратисен период след крайния срок
За да се отчетат мрежови закъснения или бавна обработка по време на последните секунди от състезанието, платформата предоставя кратък **Гратисен период** (Grace Period) веднага след официалния краен срок.
* Таймерът за обратно отброяване в интерфейса ще промени цвета си на **кехлибарен/оранжев** и ще покаже предупредителен текст, когато гратисният период е активен.
* Решения, получени след изтичането на този точен гратисен прозорец, се отхвърлят строго.
